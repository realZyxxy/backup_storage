id="4d8a"><strong class="kv ew">In most cases there are several different ways to organise your code.</strong> Criteria that defines the correct way(s) to organise your code blocks are as follows:</p><ol class=""><li class="kt ku ev kv b kw kx ky kz la lb lc ld le lf lg lh li lj lk ll lm ln lo lp lq lr ls lt bj" id="31a0">your code <strong class="kv ew">outputs correct results</strong> for any valid combination of user inputs (if there are any user inputs);</li><li class="kt ku ev kv b kw lu ky kz la lv lc ld le lw lg lh li lx lk ll lm ly lo lp lq lr ls lt bj" id="221d">there are <strong class="kv ew">no repetitions in your code</strong> (repetitions make the code long to write, long to read and impossible to maintain). This includes structural repetitions;</li><li class="kt ku ev kv b kw lu ky kz la lv lc ld le lw lg lh li lx lk ll lm ly lo lp lq lr ls lt bj" id="f612">your code is <strong class="kv ew">as simple as possible so that other people can understand your code easily. </strong>Ideally, other people should be able to modify it if needed. To do this safely, there should be some sample code, called regression tests, that they would run to make sure they had not introduced a bug by accident while doing the change;</li><li class="kt ku ev kv b kw lu ky kz la lv lc ld le lw lg lh li lx lk ll lm ly lo lp lq lr ls lt bj" id="007a">your code outputs <strong class="kv ew">intelligible error message(s)</strong> if the user input(s) are not valid;</li><li class="kt ku ev kv b kw lu ky kz la lv lc ld le lw lg lh li lx lk ll lm ly lo lp lq lr ls lt bj" id="ef84">your code <strong class="kv ew">runs reasonably quickly and consumes reasonable amount of computer memory</strong> (this criteria is almost always satisfied for the beginner’s code).</li></ol><p class="pw-post-body-paragraph kt ku ev kv b kw kx ky kz la lb lc ld le lf lg lh li lj lk ll lm ln lo lp lq eo bj" id="eec3">In addition to this, there are style guides. The official Python style guide is <a class="af lz" href="https://peps.python.org/pep-0008/" rel="noopener ugc nofollow" target="_blank">PEP 8</a>.
